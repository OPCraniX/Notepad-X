Notepad-X Help

Overview
Notepad-X is a tabbed desktop text editor focused on fast editing, search, side-by-side comparison, shared notes, and safer handling of large files. The app keeps a familiar single-window editor layout while adding session restore, Markdown preview, syntax-aware themes, language switching, encrypted saves, and recovery tools.

Quick Start
- `Ctrl+W` opens a file.
- `Ctrl+T` opens a new untitled tab.
- `Ctrl+S` saves the current tab.
- `Ctrl+F` opens the Find panel.
- `Ctrl+R` opens the Replace panel.
- `Ctrl+Q` opens compare mode.
- `Ctrl+Shift+P` opens Markdown Preview.
- `Ctrl+Shift+X` closes Help, About, Markdown Preview, compare mode, Find, or Replace before exiting the app.

Main Window
- The title bar shows Notepad-X and the active document name.
- The menu bar contains File, Edit, View, and Help.
- The tab strip holds every open document.
- The center editor shows the active tab.
- The optional right pane shows compare output or Markdown preview.
- The optional far-right sidebar shows `Currently Editing` information for the active saved file.
- The bottom status bar shows document position and state on the left and a live clock on the right.

Status Bar
- Position is shown as line and column.
- Character counts show the total document length and selection length when text is selected.
- UTF-8 is shown as the active encoding indicator.
- Zoom shows `Normal` at default size and a percentage when zoomed.
- File-backed tabs can show:
  ID: the editor identity for the current file
  Notes Synced
  unread note count with the `F3` hint
  active editor count
- Compare mode adds a status readout for the right-side compared document.
- Large-file tabs can show `Virtual` or `Preview`.
- Memory usage is shown in MB.
- The far-right clock shows day, time, and date.

Tabs
- Every document opens in its own tab.
- Untitled tabs are named `Untitled 1`, `Untitled 2`, and so on.
- Untitled numbering reuses the lowest available untitled name still open.
- Modified tabs show an asterisk after the tab name.
- Opening an already open file switches to the existing tab instead of opening a duplicate.
- Tabs can be reordered with drag and drop.
- Tabs remember caret position and scroll position when switching away and back.
- File-backed tabs are restored from the saved session on launch.
- The app keeps at least one tab open while it is running.

Line Numbers
- The editor includes a GitHub-style line number gutter.
- `View > Numbered Lines` shows or hides the gutter.
- Clicking a line number copies that full line to the clipboard.
- A small in-window notification appears beside the copied line.
- The gutter stays in sync during typing, scrolling, compare mode, and large-file virtual mode.

File Menu
- Open: Opens a single file.
- Open Project: Opens a chosen file and then other related files from the same folder.
- Grab Git: Downloads a public GitHub repository from a `username/project` entry and then opens selected files from the downloaded project.
- Recent: Reopens recently used files or clears the recent list.
- New Tab: Opens a fresh untitled tab.
- Close Tab: Closes the current tab and prompts to save if needed.
- Save: Saves the current tab.
- Save all: Saves every normal editable tab and skips large virtual or read-only tabs.
- Save As: Saves a copy of the current tab to a new path. In very large virtual or preview modes, this copies the source file stream to a new location.
- Save and Run: Saves the current tab and launches supported script or HTML files.
- Save As Encrypted: Saves a passphrase-protected `.npxe` copy.
- Print: Sends the current saved file to the Windows print action.
- Export Notes: Exports the current tab's notes to JSON or Markdown.
- Exit: Closes the app. Open panels are closed first by `Ctrl+Shift+X`.

Edit Menu
- Undo and Redo reverse or reapply edits.
- Cut, Copy, Paste, and Select All provide standard clipboard actions.
- Find opens the live Find panel.
- Find Next moves to the next match, or to the next unread note when Find and Replace are closed.
- Find Previous moves to the previous match while Find or Replace is open.
- Replace opens the Find/Replace panel.
- Date inserts the current date.
- Time/Date inserts the current time and date.
- Font opens the editor font picker.
- Edit with Notepad-X toggles the Windows Explorer context-menu entry for supported text files.
- Sound toggles note sounds.
- Language switches the visible UI language.

Editing Helpers
- Pressing `Enter` carries the current line indentation forward.
- If the text before the caret ends in `(`, `[`, `{`, or `:`, one extra indent level is added on the new line.
- Bracket matching highlights the nearest matching `()`, `[]`, or `{}` pair around the caret.
- These helpers work in the main editor and in the editable compare pane.

Autocomplete
- Autocomplete is enabled by default for normal editable tabs.
- Suggestions are built from words already present in the current tab.
- The suggestion list opens below the caret while typing.
- `Up` and `Down` move through suggestions.
- `Tab` or `Enter` accepts the selected suggestion.
- `Esc` closes the suggestion list.
- `View > Autocomplete` shows or hides autocomplete.

Spell Check
- Spell Check can underline misspelled words in normal editable text-oriented tabs.
- Right-clicking a marked word can show replacements and `Add to Dictionary`.
- `View > Spell Check` or `F7` toggles the checker.
- Spell Check stays out of buffered large-file tabs, preview tabs, and most syntax-highlighted code tabs.

Find, Replace, and Find In Files
- Find highlights matches live while typing.
- Live find does not move the caret during typing.
- Pressing `Enter` in the Find or Replace query box jumps to the first match from the top of the active editor pane.
- After that jump, the caret lands at the end of the found match.
- The Find panel's `Find` button and `F3` continue forward through matches.
- `Shift+F3` moves backward through matches.
- Replace uses the same live highlighting in its Find field.
- Forward and backward search continue across open tabs by default.
- The Find panel also includes a `Find In:` query field and a `Browse` button.
- Type the folder-search text into `Find In:`.
- Match highlighting follows the visible query across open tabs, including after switching tabs.
- `Browse` chooses the folder for Find In Files without replacing the text in the `Find In:` box.
- Pressing `Enter` in `Find In:` searches the most recently selected folder or prompts for one if none has been chosen yet.
- Find In Files searches supported files in the selected folder and shows a results window with match counts and an `Open Selected` action.
- In compare mode, search commands follow the editor pane clicked most recently.
- Find is not available in buffered large-file virtual mode.
- Replace is not available in buffered large-file virtual mode.

Markdown Preview
- `View > Preview Markdown` or `Ctrl+Shift+P` opens a live rendered Markdown preview in the right pane.
- The preview updates while the source tab changes.
- Headings, lists, quotes, divider rules, fenced code blocks, bold, italic, inline code, and links are rendered.
- Opening Markdown Preview closes compare mode first.

Compare Mode
- `View > Compare Tabs` or `Ctrl+Q` opens compare mode.
- The active tab stays on the left and the selected comparison tab opens on the right.
- Normal tabs remain editable in both panes.
- The compared file shows its own bottom status readout.
- `Currently Editing` remains on the far right if that sidebar is open.
- Opening or closing the sidebar re-centers the compare split.
- Search commands follow the compare pane clicked most recently.
- `Ctrl+Shift+X` closes compare mode.
- If compare mode is open when the app closes, the same pair can be restored on the next launch when both files still exist.

Currently Editing
- `View > Currently Editing` opens a right-side sidebar for saved files.
- The sidebar shows editors currently tracked on the active file.
- The sidebar remains separate from compare mode and Markdown preview.

Notes
- Notes attach to selected text through the context menu.
- Add note is unavailable when no text is selected.
- Notes include:
  author name
  note text
  note color
- Available note colors are Yellow, Green, Red, and Light Blue.
- Clicking a highlighted note opens the note popup.
- The popup shows note details, timestamps, and responses.
- Existing notes support `Respond` and `Remove note`.
- The latest response color becomes the current highlight color.
- Note data is stored beside the document so multiple editors can share it.

Unread Notes and Navigation
- `F3` jumps to the oldest unread note when Find and Replace are closed.
- Each `F3` press continues through unread notes from oldest to newest.
- `F4` cycles all notes in the current tab and wraps at the end.
- `View > Filter Notes` limits note cycling to All, Unread, Yellow, Green, Red, or Light Blue.
- Opening a note by clicking it clears its unread state for the current editor identity.

Note Export
- Export Notes writes the current tab's notes to:
  JSON for machine-readable export
  Markdown for a readable report

Audio
- A sound can play when a new unread note arrives for the active file.
- A second sound can play when a note is removed.
- `Edit > Sound` toggles sound playback.

Themes and Syntax
- Built-in themes are available from `View > Syntax Theme`.
- `Create Theme` opens a color-picker workflow for building a new theme.
- Newly created themes appear in the theme list immediately.
- `View > Syntax Mode` overrides automatic syntax detection for the active tab.
- Available syntax overrides include Auto, Plain Text, Python, C, C++, Rust, Java, JavaScript, HTML, PHP, XML, and SQL.

Language Support
- `Edit > Language` lists available UI languages.
- Friendly native language names are shown when available.
- The selected language is saved in the session and restored on launch.
- Arabic switches the visible UI text direction to right-to-left.
- Locale changes can also switch the editor font to a better installed script-friendly fallback automatically.

Save and Run
- Save and Run saves the current tab before launching it.
- Supported launch targets include Python, JavaScript, PHP, batch, PowerShell, shell, and HTML.
- HTML opens in the default browser.
- Script files launch in a new console window when a matching runtime is available.
- Save and Run can also use the current Syntax Mode for Python, JavaScript, PHP, and HTML tabs without the usual file extension.
- Save and Run is unavailable in buffered large-file or preview tabs.

Encrypted Files
- `Save As Encrypted` creates `.npxe` encrypted copies.
- Encrypted saves suggest a `.npxe` file name automatically.
- Opening an encrypted file prompts for the passphrase.
- Normal `Save` and `Save As` remain plain-text save flows.

Large Files
- Large files load in the background to keep the UI responsive.
- Large text files remain editable up to a practical limit.
- Extremely large files can fall back to buffered virtual mode.
- Binary-like files are treated more cautiously and can open in a safer preview-style mode.
- In virtual mode:
  editing is disabled
  direct saving is disabled
  Find and Replace are disabled
  Go To Line still works against the full file
  scrolling and selection are limited to the loaded buffer window
- `Save As` remains available for copying the source file elsewhere.

Saving and File Safety
- Normal saves use an atomic temp-file replacement pattern.
- Session and note support files are also written atomically.
- Save all keeps the previously active tab selected when it finishes.
- A warning appears before overwriting a file that changed on disk after it was opened.
- Recovery restores content into tabs instead of overwriting existing user files directly.
- Permissions problems are shown to the user instead of failing silently.

Session Restore and Recovery
- File-backed tabs are written to the session and restored on startup.
- The last active tab is restored when possible.
- Compare mode can be restored when both files still exist.
- Sound, status bar, numbered lines, autocomplete, spell check, Markdown Preview, compare page-sync, and theme settings are restored.
- Unsaved untitled tabs are not part of normal session restore, but they can be recovered after a crash.
- Modified file-backed tabs can also be restored after a crash without overwriting the on-disk file automatically.
- If recovery data is found after a previous crash, the app offers to restore it.

Zoom
- `Ctrl+Mouse Wheel` zooms in and out.
- `Ctrl++` zooms in.
- `Ctrl+-` zooms out.
- Zoom affects the editor font size within the configured limits.

Printing
- Print uses the Windows file print action on the saved file path.
- If the current tab has never been saved, the app prompts for a save first.
